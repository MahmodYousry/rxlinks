# Project structure

Where things live in the Rxlinks project (simplified).

---

## Top level

| Item | What it is |
|------|------------|
| `src/` | All app source code. |
| `public/` | Static files: favicon, images, and **links.json** (the list of links). |
| `docs/` | Documentation (the files you’re reading). |
| `angular.json` | Angular project config (build, serve, test). |
| `package.json` | Project name, scripts, and dependencies (npm packages). |

---

## Inside `src/`

| Path | What it is |
|------|------------|
| `index.html` | The single HTML page that loads the app. |
| `main.ts` | Entry point: boots the Angular app. |
| `styles.scss` | Global styles (e.g. body, theme light/dark). |
| `app/` | The Angular app (components, routes, services). |

---

## Inside `src/app/`

| Path | What it is |
|------|------------|
| `app.ts` | Root component: just shows the router outlet. |
| `app.routes.ts` | Routes: `/` → redirects to main-links; `/main-links` → main page. |
| `app.config.ts` | App-wide config (router, PrimeNG, etc.). |
| `components/` | UI pieces: **main-links** (main page), **search**, **link** (one card). |
| `Shared/` | **LinksService**: loads links from `links.json`. |
| `models/` | Types and enums (e.g. link shape, theme light/dark). |

---

## Inside `public/`

| Path | What it is |
|------|------------|
| `links.json` | **The list of all links.** Edit this file to add/remove/change links. |
| `imgs/app-icons/` | Icon images for each link. |
| `favicon.ico` | Browser tab icon. |

---

## You don’t need to touch

- `node_modules/` – installed packages (don’t edit).
- `dist/` – build output (generated by `ng build`).
- `*.spec.ts` – unit test files (unless you’re writing tests).

To **change what links appear**, you only need to edit **`public/links.json`** (and add images in `public/imgs/app-icons/` if needed). See [Editing links](05-editing-links.md).
